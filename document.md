# JRFirstGame 技术文档

## 1. 项目概述

JRFirstGame 是一个基于 C++17 + SFML 3.x + SQLite3 构建的 RPG 闯关游戏。玩家在无限地图中击杀敌人、收集掉落物、提升属性，通过阶段性胜利不断强化角色。支持单机闯关和双人UDP联机合作两种模式。

### 1.1 核心玩法

- **单机闯关**：选择角色后进入无限地图，击杀敌人获得分数和经验，分数达到阈值触发阶段胜利，角色获得职业专属强化
- **双人闯关**：一人创建房间（主机），另一人加入（客户端），共享同一地图合作对抗敌人，击杀分数共享
- **角色成长**：六大属性随机生成，职业提供属性加成，升级增加全属性，掉落物可永久提升属性，分数提供倍率加成

### 1.2 技术选型

| 技术 | 版本 | 用途 |
|------|------|------|
| C++ | 17 | 核心语言标准，使用 std::optional、std::string_view 等特性 |
| SFML | 3.x | 图形渲染、窗口管理、网络通信 |
| SQLite3 | 3.x | 用户账户、角色数据、排行榜的持久化存储（源码编译） |
| CMake | 3.15+ | 跨平台构建系统 |
| GitHub Actions | - | 自动构建 Windows/Linux/macOS 并发布 Release |

## 2. 架构设计

### 2.1 整体架构

```
┌─────────────────────────────────────────────────┐
│                   main.cpp                      │
│         主循环：事件→更新→绘制，视图切换          │
└──────────────────────┬──────────────────────────┘
                       │
          ┌────────────┴────────────┐
          │      View（基类）        │
          │  handleEvent/update/draw │
          └────────────┬────────────┘
                       │
    ┌──────────┬───────┼───────┬──────────┐
    │          │       │       │          │
 MainMenu  AuthView  Menu   GameView  OnlineView
                        │               │
                   CharacterView    RankingsView
```

### 2.2 视图切换机制

主循环通过 `std::unique_ptr<View>` 管理当前视图。视图切换通过回调函数实现：

```cpp
std::function<void(std::string)> changeView = [&](std::string name) {
    pendingView = name;
};
```

每帧检查 `pendingView`，根据字符串创建对应的视图对象。支持的语言切换会设置 `languageChanged` 标记，主循环检测后重建当前视图以刷新所有文字。

## 3. 核心系统详解

### 3.1 属性系统

六大属性定义在 `Attributes` 结构体中：

| 属性 | 英文 | 影响 |
|------|------|------|
| 力量 | strength | 剑伤害加成（strength * 0.8） |
| 敏捷 | agility | 刺刀伤害加成（agility * 0.8） |
| 魔法 | magic | 法器伤害加成（magic * 0.8） |
| 智力 | intelligence | 预留扩展 |
| 生命 | vitality | 血量上限 = 120 + vitality * 8 |
| 幸运 | luck | 预留扩展 |

**属性计算链**：
```
最终属性 = 基础属性 + 职业加成 + 等级加成 + 分数加成
```

- 基础属性：创建角色时随机生成（正态分布，均值10，标准差4，范围[1,20]，总和上限80）
- 职业加成：战士力量+5，法师魔法+5，刺客敏捷+5
- 等级加成：每级每项+2，即 2*(level-1)
- 分数加成：每50分每项+1，上限20

### 3.2 职业系统

| 职业 | 属性加成 | 武器 | 攻击方式 | 阶段强化方向 |
|------|---------|------|---------|-------------|
| 战士 | 力量+5 | 铁剑 | 扇形近战 | 伤害+5/阶段，攻速+15%/阶段，攻击范围增大 |
| 法师 | 魔法+5 | 火焰法器 | 远程投射物 | 伤害+4/阶段，爆炸半径+30/阶段，攻速+10%/阶段 |
| 刺客 | 敏捷+5 | 暗影刺刀 | 扇形近战（快速） | 伤害+3/阶段，暴击率+12%/阶段，暴击倍率递增，攻速+25%/阶段 |

### 3.3 等级系统

- 升级经验需求：`100 * 当前等级`
- 升级后经验溢出自动继续判断
- 等级上限：999

### 3.4 战斗系统

**伤害计算**：
```
总伤害 = (武器基础伤害 + 属性加成 + 阶段强化伤害) × 分数倍率 × 暴击倍率
```

- 属性加成：剑→力量*0.8，法器→魔法*0.8，刺刀→敏捷*0.8
- 分数倍率：1.0 + min(总分数/50 * 0.1, 1.0)，范围[1.0, 2.0]
- 暴击：基础5%概率，刺客阶段强化后最高80%，暴击倍率1.5起

**攻击方式**：
- 近战（战士/刺客）：180度扇形范围判定，显示刀光动画
- 远程（法师）：向最近敌人发射投射物，命中后可爆炸范围杀伤

**翻滚系统**：
- 持续时间0.3秒，冷却0.8秒
- 翻滚期间无敌（isInvincible=true），半透明显示
- 翻滚方向由当前WASD输入决定

### 3.5 敌人系统

| 敌人 | 血量 | 伤害 | 速度 | 攻击方式 | 击杀分数 | 颜色 |
|------|------|------|------|---------|---------|------|
| 史莱姆 | 40 | 8 | 50 | 近战追踪 | 15 | 绿色 |
| 骷髅刀斧手 | 80 | 15 | 80 | 近战追踪 | 30 | 灰白色 |
| 骷髅弓箭手 | 50 | 12 | 70 | 远程射击（保持150-250距离） | 25 | 暗黄色 |
| 巨人 | 200 | 30 | 25 | 近战追踪 | 60 | 棕色 |

**难度缩放**：
```
敌人缩放因子 = 地图难度 × 时间难度 × 分数难度
```
- 地图难度：1.0 + 已探索区域数 * 0.05
- 时间难度：1.0 + 游戏时间/600
- 分数难度：1.0 + 总分数/1000

缩放后：血量/伤害/分数乘以缩放因子，速度乘以sqrt(缩放因子)。

### 3.6 掉落物系统

敌人被击杀后概率掉落：

| 掉落物 | 概率权重 | 效果 | 颜色 |
|--------|---------|------|------|
| 血瓶 | 30% | 回复33%血量 | 红色 |
| 力量提升 | 15% | 力量+2（上限30） | 橙色 |
| 敏捷提升 | 15% | 敏捷+2 | 青色 |
| 魔法提升 | 15% | 魔法+2 | 紫色 |
| 生命提升 | 13% | 生命+2 | 绿色 |
| 范围提升 | 12% | 攻击范围+15 | 黄色 |

基础掉率40%，强敌掉率更高（巨人80%，骷髅刀斧手55%，骷髅弓箭手45%）。
掉落物10秒后消失，带有上下浮动动画和光晕效果。

### 3.7 阶段胜利系统

分数达到以下阈值时触发阶段胜利：

| 阶段 | 分数阈值 | 效果 |
|------|---------|------|
| 1 | 100 | 职业专属强化等级1，回满血 |
| 2 | 1000 | 职业专属强化等级2，回满血 |
| 3 | 2000 | 职业专属强化等级3，回满血 |
| 4 | 5000 | 职业专属强化等级4，回满血 |
| 5 | 10000 | 最终胜利，职业专属强化等级5，回满血 |

### 3.8 地图系统

- 世界被划分为 800×800 像素的区域网格
- 玩家进入新区域时，检查周围3×3范围并生成敌人
- 敌人配置由区域到原点的曼哈顿距离决定：
  - 距离0：1只史莱姆
  - 距离1-2：史莱姆+骷髅刀斧手
  - 距离3-4：史莱姆+骷髅刀斧手+骷髅弓箭手
  - 距离5-6：2骷髅刀斧手+骷髅弓箭手
  - 距离7-8：2骷髅弓箭手+巨人
  - 距离9+：巨人+骷髅刀斧手+骷髅弓箭手（数量随距离增长）
- 摄像机始终跟随玩家
- 背景绘制半透明网格线标识区域边界

## 5. 网络系统

### 5.1 架构

采用主机-客户端架构：

- **主机**：运行完整游戏逻辑，接收客户端输入，广播游戏状态
- **客户端**：发送输入和攻击信号，接收并渲染主机同步的状态

通信使用 UDP 协议，SFML UdpSocket 实现，非阻塞模式。

### 5.2 消息协议

| 消息类型 | 方向 | 数据内容 |
|---------|------|---------|
| Connect | C→S | 职业类型(int) + 用户名(string) |
| ConnectAccept | S→C | 分配的玩家ID(int) |
| PlayerInput | C→S | 玩家ID + 上下左右4个bool |
| PlayerAttack | C→S | 玩家ID |
| PlayerRoll | C→S | 玩家ID + 方向x/y |
| GameState | S→C | 玩家列表 + 敌人列表 + 投射物列表 + 掉落物列表 |
| GameStart | S→C | 无 |
| PauseSync | 双向 | 暂停状态bool |
| Disconnect | 双向 | 玩家ID |

### 5.3 同步数据结构

**GameState 包格式**：
```
[玩家数量]
  每个玩家: ID, posX, posY, health, maxHealth, totalScore, level, name
[敌人数量]
  每个敌人: type, posX, posY, health, maxHealth, radius, score
[投射物数量]
  每个投射物: posX, posY, dirX, dirY, speed, damage, maxDistance, traveled, fromPlayer, colorR, colorG, colorB
[掉落物数量]
  每个掉落物: type, posX, posY, lifetime, bobTimer
```

### 5.4 频率控制

- 输入发送：~30Hz（每33ms发送一次）
- 攻击信号：~10Hz（每100ms发送一次）
- 状态广播：每帧（60Hz）
- 网络接收：每帧最多处理10个包

## 6. 武器系统

### 6.1 C语言链表层（WeaponList.c）

纯C实现的单链表，提供创建/销毁/插入/删除/查找/遍历接口。使用 `extern "C"` 保证C++可调用。

### 6.2 C++桥接层（WeaponListBridge）

RAII封装C链表，提供类型安全接口：
- 构造时调用 `weapon_list_create()`
- 析构时调用 `weapon_list_destroy()`
- `cToCpp()` 静态方法将C Weapon节点转为C++ WeaponData
- 禁止拷贝，允许移动语义

### 6.3 默认武器

| 武器 | 类型 | 伤害 | 范围 | 攻速 | 远程 |
|------|------|------|------|------|------|
| 铁剑 | WEAPON_SWORD | 15 | 80 | 2.5 | 否 |
| 火焰法器 | WEAPON_ARTIFACT | 20 | 300 | 0.8 | 是 |
| 暗影刺刀 | WEAPON_DAGGER | 8 | 50 | 4.0 | 否 |

## 7. 国际化系统

单例模式，哈希表存储翻译键值对。每个键对应 `{英文, 中文}` 一对翻译。

默认语言为中文，可通过 L 键或右上角按钮切换。切换后设置 `languageChanged` 标记，主循环重建当前视图以刷新所有文字。

翻译覆盖范围：菜单、登录/注册、角色创建/选择、游戏内UI、排行榜、联机大厅、武器名、敌人名等。

## 8. 安全相关

### 8.1 密码存储

- 注册时生成16字节随机盐值（32字符16进制）
- 密码哈希：SHA-256 + 盐值，10000轮迭代（类PBKDF2）
- 哈希过程：`SHA256(salt + password + salt)` → 9999轮 `SHA256(上一轮结果 + salt + password + salt)`
- 存储格式：64字符16进制字符串

### 8.2 SQL注入防护

所有数据库查询使用参数化语句（`sqlite3_bind_text`/`sqlite3_bind_int`），不拼接SQL字符串。

## 9. 构建与发布

### 9.1 CMake配置

- C++17标准，UTF-8编码
- SQLite3源码编译为静态库
- 可选字体嵌入（EMBED_RESOURCES=ON，默认开启）
- 跨平台SFML查找（支持3.x和2.x回退）
- 平台特定链接：Linux需X11/OpenGL，Windows需winmm/ws2_32，macOS需CoreFoundation

### 9.2 GitHub Actions CI/CD

工作流 `.github/workflows/build.yml`：

1. 推送到main/master分支时触发构建
2. 推送v*标签时触发构建+发布Release
3. 三个平台并行构建：Windows(VS2022)、Linux(GCC)、macOS(Clang arm64)
4. 自动下载SFML 3.0.2预编译包
5. 构建后打包：Windows→zip，Linux→tar.gz，macOS→app zip
6. 发布时上传三个平台的包到GitHub Release

## 10. 渲染细节

### 10.1 玩家渲染

- 圆形主体（半径18），翻滚时半透明（alpha=128）
- 上方血条（50×8像素，红底绿条）
- 名字显示在下方
- 武器图标显示在右侧（剑=银白长条，法器=橙色圆，刺刀=紫色短条）

### 10.2 敌人渲染

- 45度旋转正方形（菱形）
- 上方血条
- 名字带黑色描边（4方向偏移）提高可读性

### 10.3 刀光动画

- 180度扇形弧线扫过
- 24段梯形四边形组成
- 剑尖处最亮，拖尾渐隐
- 剑尖高亮光点
- 颜色按职业设置（战士/法师/刺客各有不同橙色色调）

### 10.4 伤害数字

- 普通伤害：黄色18号字
- 暴击伤害：红色24号字+描边
- 环绕敌人位置缓慢旋转
- 2秒后消失，最后0.6秒淡出

### 10.5 掉落物渲染

- 外圈光晕（半径12，半透明）
- 内圈（半径8，实色）
- 中心高亮点（半径3，白色）
- 上下浮动动画（sin波，振幅3像素）

## 11. 已知限制

1. 联机模式仅支持局域网UDP，无断线重连和延迟补偿
2. 密码哈希使用djb2算法，不可用于生产环境
3. macOS版本未经深度测试
4. 客户端不运行游戏逻辑，完全依赖主机同步，主机断开则游戏结束
5. 掉落物拾取仅在主机端判定，客户端看到的是同步后的效果
6. 敌人数量无上限，极端情况下可能影响性能
